## Commands

```bash
# Local dev server (port 5599)
python3 -m http.server 5599

# Build for production (minifies JS+CSS → dist/)
npm run build

# Deploy — Vercel auto-builds on push
git push
```

Build uses `terser` + `clean-css-cli` (local devDependencies). If `npm run build` fails due to npm cache permissions, fix with:
```bash
npm install --cache /tmp/npm-cache
```

**Never edit `dist/`** — it's rebuilt on every deploy and is gitignored.

## Architecture

Single-page app. No framework, no bundler at runtime. Three source files:

| File | Size | Role |
|---|---|---|
| `mkt201_midterm.html` | ~5,000 lines | Full HTML shell — all pages as `<div class="page" id="page-*">` toggled by `showPage()` |
| `mkt201_scripts.js` | ~6,400 lines | All data arrays + all logic |
| `mkt201_styles.css` | ~5,200 lines | All styles, dark mode via `body.dark`, Arabic toggle via `body.arabic-hidden` |

`index.html` is a redirect to `mkt201_midterm.html` (generated by build.js).

## Chapter Content Structure (HTML)

Each chapter page (`page-ch1`, `page-ch2`, `page-ch3`, `page-ch5`) contains:
- `<div class="chapter">` blocks, each with an `<h2>` Learning Objective header
- `<div class="lo-section">` blocks with `.lo-header` + `.lo-body`
- Content inside `.lo-body` uses these CSS classes:

| Class | Purpose |
|---|---|
| `def-spotlight` | Key definition box (contains `.def-term` + `.def-body`) |
| `concept-row` + `concept-card teal/blue/purple/amber/green/rose` | Side-by-side concept cards |
| `example-box` | Real-world example block |
| `from-notes-wrap` | Wraps notes-sourced content; first child must be `<span class="from-notes-badge">📝 من النوت</span>` |
| `memory-box` | Quick recall / memory aid |
| `exam-tip` | Exam hint |
| `slide-bullets` | Styled `<ul>` for bullet points |
| `ref-table` | Comparison/reference tables |

Arabic translations go inside `.ar-line` spans/divs. The CSS rule `body.arabic-hidden .ar-line { display:none }` hides all Arabic when the toggle is off. Default is Arabic visible (`arabicVisible = true`).

## PDF Export (`exportChapterBinder`)

Reads directly from the live DOM of the target page. Handles: `def-spotlight`, `concept-row`, `concept-card`, `example-box`, `from-notes-wrap`, `memory-box`, `exam-tip`, `process-flow`, `step-card`, `ref-table`, `h3`, `h4`, `ul`, `ol`, `p`, and generic divs (recursive fallback). Unrecognized elements are silently skipped.

**Critical rule:** Do NOT add quiz questions (`tb_questions.js` or `allQuestions[]`) when working on chapter content. Content work and question bank work are separate tasks.

## Question Data Format

```js
{ ch:"ch1", diff:"easy", q:"Question text",
  opts:["A","B","C","D","E"], ans:2,  // 0-indexed
  exp:"Explanation (Arabic)",
  expW:{0:"why A wrong", 1:"why B wrong"},  // omit correct index
  past:true  // optional
}
```

`pastQuestions[]` (lines ~680–697) has no inline `ch` — assigned by IIFE using `chMap[]`.
